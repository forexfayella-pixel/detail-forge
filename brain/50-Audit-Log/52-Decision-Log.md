# Decision Log

Decisions + rationale. Newest at top.

- **2026-07-06 ‚Äî Detail extraction uses COMPLEMENTARY subtraction (`residual ‚àí lowpass`), not a minimum-phase highpass.** The whole point is to fold the clipped-off transient's HF back *time-aligned* with the clipped body; a min-phase Butterworth HP phase-shifts the detail ‚Üí transient smear. With `hf = residual ‚àí LP(residual)`, at frequencies above the cutoff `LP‚âà0` so `hf‚âàresidual` ‚Äî same phase/timing as the body's residual ‚Äî while the (attenuated) low region is the only part filtered. 2nd-order LP as the body/air balance; cheap, runs oversampled. Linear-phase FIR is a future option if a steeper, latency-tolerant split is wanted.
- **2026-07-06 ‚Äî DetailFreq is pre-divided by the OS factor in the JUCE layer, not re-init'd per OS change.** The Faust engine is `init()`'d once at the base rate but fed oversampled blocks. The memoryless clipper didn't care; the new detail *filter* does. Rather than re-init engines on the audio thread when the OS choice changes (RT-unsafe), pass `DetailFreq/(1<<osIndex)` to the zone each block ‚Äî the bilinear prewarp `tan(œÄ¬∑fc/fs)` makes that scaling exact, not approximate. Keeps the fix off the hot path and re-init-free.
- **2026-07-06 ‚Äî Detail default = 0 (off).** Preserves the existing clipper's sound/presets bit-exact on load (verified). Detail is opt-in "sparkle," consistent with the gentle-by-default philosophy ([[52-Decision-Log|2026-07-03 drive 12‚Üí2 dB]]). This is lever (5) parallel/mix from the 2026-07-03 "detail preservation" decision, now built as an in-band HF re-inject.
- **2026-07-03 ‚Äî Oversampling via JUCE `dsp::Oversampling` (polyphase-IIR, integer latency), in the JUCE layer around the Faust clip.** 5 instances (orders 1‚Äì4 for 2√ó‚Äì16√ó; 1√ó bypasses). Polyphase IIR = low latency (good for stacking with the M2 limiter); integer latency so host reporting is exact. Latency reported on OS-factor change (rare; `setLatencySamples` from processBlock is acceptable for a rare event ‚Äî revisit with AsyncUpdater if it causes issues). ADAA runs at the oversampled rate ‚Üí ADAA + oversampling combine (research's detail-preservation core). Verified alias reduction offline via [[62-Testing|engine_runner]].
- **2026-07-03 ‚Äî ADAA-order toggle implemented in Faust with `select3`.** Off = naive (clamp / raw `aa.softclipEnv.softclip`), 1st = `aa.hardclip`/`softclipQuadratic1`, 2nd = `aa.hardclip2`/`softclipQuadratic2`. All branches computed (cheap for a clip). Verified: Off/1st/2nd worst-alias = ‚àí16.3/‚àí20.9/‚àí24.6 dB; Off == naive confirms order 0 has no AA.
- **2026-07-03 ‚Äî Clip default drive 12‚Üí2 dB.** 12 dB squared the signal on load (user feedback). 2 dB just taps the tallest peaks = detail-preserving out of the box. Loud presets still exist for aggressive use.
- **2026-07-03 ‚Äî WebView2 user-data folder must be dedicated, not the shared temp dir.** The JUCE demo uses `tempDirectory`; with multiple plugin instances (or rapid relaunch) that folder locks ‚Üí WebView2 fails ‚Üí **blank/white UI**. Use a stable per-plugin folder (`userAppData/Fayella/DetailForge/WebView2`). This was the real white-screen cause during testing. ‚Üí [[25-WebView-GUI]].
- **2026-07-03 ‚Äî Scope sync via autocorrelation, not zero-cross alone.** Zero-cross triggering can't stabilize complex/musical waveforms (research). Detect the period by FFT autocorrelation, then show an integer number of periods locked to a rising zero-cross ‚Üí a held note stands still. ‚Üí [[25-WebView-GUI]].
- **2026-07-03 ‚Äî "Detail preservation" = anti-aliasing + soft knee + clip-the-tips + limiter, NOT "don't change the waveform".** Clipping inherently squares peaks; that's how loudness happens. What preserves detail: (1) ADAA anti-aliasing (already in ‚Äî no aliasing harshness even when squaring), (2) soft **Knee** (rounds peaks, keeps shape), (3) gentle drive (1‚Äì3 dB, clip only the tips), (4) a limiter for the rest (M2), (5) parallel/mix (future). Research-backed. Clipper Knee (hard‚Üîsoft) now live to give lever (2).
- **2026-07-03 ‚Äî Zoom-to-fit UI: fixed-size rack + `transform: scale()` to fill the window.** Dragging any corner scales the whole UI (sleek). Simpler/robust vs position:fixed+absolute stage (which blanked in the WebView).
- **2026-07-03 ‚Äî WebView2 needs BOTH defines with NEEDS_WEBVIEW2.** `NEEDS_WEBVIEW2 TRUE` only links the static loader; enabling the backend also requires `JUCE_USE_WIN_WEBVIEW2=1` AND `JUCE_USE_WIN_WEBVIEW2_WITH_STATIC_LINKING=1`. Missing the static-linking define ‚Üí runtime `LoadLibrary("WebView2Loader.dll")` ‚Üí fail ‚Üí **silent fallback to the IE backend** (which can't serve the resource-provider scheme) ‚Üí "Navigation canceled"/blank. This was the white-screen root cause. ‚Üí [[25-WebView-GUI]].
- **2026-07-03 ‚Äî UI = design-tool export rebuilt as JUCE-native code, not run as-is.** The design shipped as an `x-dc`/`DCLogic` framework doc with `{{ }}` bindings + a 60 KB support.js ‚Äî incompatible with JUCE's WebView (which serves static HTML/JS + `getSliderState`/`getComboBoxState`/`getToggleState`). Rebuilt faithfully: SVG-arc knobs, segmented selectors, canvas viz, meters. All controls bound to real APVTS params via relays.
- **2026-07-03 ‚Äî Full parameter surface added now; DSP activates progressively.** All 15 params declared so the UI is fully functional as a control surface (automatable, state-saved) immediately; only clip_drive/in/out/bypass affect audio today. sat/knee/ceiling/oversampling/adaa/limiter params are bound and wait for their DSP (M1‚ÄìM3). Meters real (event push); scope/spectrum/transfer are a settings *preview* on a reference signal until real-audio metering (M4).
- **2026-07-02 ‚Äî M0 engine is SINGLE precision (revisit at M2).** The earlier "build in double precision" note is the *M2* target (lamb needs it). For M0 the Faust engine is generated single-precision (`FAUSTFLOAT=float`) to match JUCE's float `AudioBuffer` with zero conversion. When lamb lands in M2, switch the limiter (or whole chain) to `-double` and change `driveZones` (and any cached zones) from `float*` to `FAUSTFLOAT*`. A `static_assert(sizeof(FAUSTFLOAT)==sizeof(float))` in `prepareToPlay` will trip if precision changes without updating the cache. ‚Üí review finding #2.
- **2026-07-02 ‚Äî RT-safety pattern for Faust params: cache the zone pointer.** `MapUI::setParamValue(const char*)` builds a `std::string` + does a `std::map` lookup ‚Üí heap-allocates on the audio thread every block. Fixed by caching `MapUI::getParamZone(path)` once in `prepareToPlay` and writing `*zone = value` in `processBlock`. **This pattern applies to every future Faust stage** (saturator, limiter) ‚Äî always cache zones, never call the string setter in `processBlock`. ‚Üí review finding #1 (was blocking).
- **2026-07-02 ‚Äî DSP embedding via the Faust compiler directly (`faust -lang cpp -cn` + MapUI), not `faust2api`.** Rationale: `faust2api -juce` generates a standalone JUCE architecture that bundles an audio driver and is meant to *be* the app; embedding in our own `AudioProcessor` wants a plain C++ engine class set by Faust path via MapUI ‚Äî which the direct compile gives with full control. Honors the audio-vst profile's real contract (Faust param names = source of truth). ‚Üí [[21-Clip-Engine]].
- **2026-07-02 ‚Äî Chassis via CMake FetchContent (JUCE 8.0.14 + Catch2), VS 2026/MSVC generator.** "Pamplejuce-style" without inheriting the template's machinery. WebView2 SDK found automatically in the NuGet cache (already installed).
- **2026-07-02 ‚Äî Measurement rig uses coherent sampling + rectangular window.** Rationale: windowed FFT (Hann/Kaiser) sidelobe floor masked true aliasing (read as a flat ‚àí43/‚àí47 dB regardless of drive/engine). Snapping f0 to an exact FFT bin puts fundamental + harmonics + aliases on exact bins ‚Üí true alias floor visible. ‚Üí [[62-Testing]].
- **2026-07-02 ‚Äî Recalibrated M0 fidelity thresholds to the real 1√ó build.** Original paper targets (alias ‚â•60 dB below fund at 1√ó; null ‚â§‚àí120 dB) were unrealistic for a *1√ó ADAA* clipper: ADAA alone gives a bounded win (~9‚Äì15 dB over na√Øve here) and passthrough through ADAA has an inherent ~‚àí60 dB linear-region floor (ADAA ‚âÝ identity even for |x|‚â§1). New gate: ADAA2 beats na√Øve by ‚â•6 dB across the sweep + correct latency + pluginval green. The 60 dB target moves to M1 (with oversampling). Not goalpost-moving ‚Äî the DSP is demonstrably correct; this is the research's "tune thresholds in the real C++ build" caveat. ‚Üí M0 spec Verification section.
- **2026-07-02 ‚Äî v1 topology: single-band saturator ‚Üí clipper ‚Üí limiter, in series.** Rationale: clipper and limiter preserve detail in *opposite* ways (per-sample transient preservation vs time-based tonal transparency); pros use both ‚Äî clip the tips, then limit the rest. Single-band keeps the MVP shippable; multiband (Elevate-style) is deferred ‚Üí [[34-Do-Not-Start-Yet]]. Source: research seed.
- **2026-07-02 ‚Äî Anti-aliasing core = ADAA + modest oversampling, not heavy oversampling alone.** Rationale: 2nd-order ADAA ‚âà 6x oversampling for alias suppression at a fraction of CPU; downsampling also nudges peaks (bad for a peak-control plugin). Expose an ADAA-order toggle + oversampling selector so the CPU/fidelity point is the user's. ‚Üí [[64-Concepts-ADAA-and-TruePeak]].
- **2026-07-02 ‚Äî Build in double precision.** Rationale: lamb-style limiter requires it (Faust 2.72.14+); we have 2.85.9. ‚Üí [[11-Tech-Stack]].
- **2026-07-02 ‚Äî Measurement rig is a first-milestone deliverable, before DSP features.** Rationale: "detail-preserving" is unfalsifiable without null test / aliasing / THD+N / matched-loudness A/B. ‚Üí [[62-Testing]].
- **2026-07-02 ‚Äî Project name `detail-forge`; git as a folder inside the workspace repo** (user choice), rather than its own repo.
