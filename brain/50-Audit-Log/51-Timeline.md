- **2026-07-06** — **Detail-preserving fold-back added to the clipper (the literal Au5 technique).** Studied Verdant's *Cruncher* (binary-only release; source `berikai/cruncher` since pulled) — its `INFO.txt` + the params I pulled from the `.clap` (`input_gain`/`threshold`/`drive`/`*_cutoff`/`output_gain`/detail aux) confirm the method Au5 [demoed on TikTok](https://www.tiktok.com/@au5_official/video/7632234833195863326): clip the body, harvest the **HF of the clipped-off residual**, fold it back. Implemented in `clip.dsp`: `residual = driven − clipped`; `hf = residual − lowpass(2, DetailFreq, residual)` (**complementary** subtraction, so above the split `hf == residual` sample-for-sample → the re-injected transient stays time-aligned with the body, no phase smear — chosen over a min-phase HP); `out = clipped + Detail·hf`. **Detail=0 collapses to the old clipCore bit-exact** (back-compat). Two new APVTS params: **Clip Detail** (0–200 % → Faust 0–2) and **Clip Detail Freq** (300 Hz–12 kHz, log-skewed, default 3 kHz). Runs **inside the oversampled region** so the boosted HF is anti-aliased; the engine is init()'d at base rate so the JUCE layer **pre-divides DetailFreq by the OS factor** (`1<<osIndex`) — exact under the bilinear prewarp. Regenerated committed `ClipEngine.h` (`regen_faust`). **Verified** with a standalone harness driving the real engine (impulse-on-body): Detail=0 deterministic + bit-exact vs plain; Detail=1 restores **47× HF energy** above 3 kHz while LF body RMS moves **+2.8 %**; incremental **DetailForge build clean**. **UI wired same session:** added `clip_detail`/`clip_detail_freq` to `kSliderIds` (relay/attachment loops bind them automatically) + a second knob row in the CLIPPER panel (**DETAIL** %, **D-FREQ** kHz with a new `hz` formatter); made the JS `setSliderScaled` + dblclick-reset **skew-aware** (`norm = pow(proportion, skew)`, skew defaults to 1 so linear params are untouched) so the skewed freq knob recalls correctly; INIT + 3 aggressive presets seeded with detail.
